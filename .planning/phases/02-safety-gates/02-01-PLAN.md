**Function 1: Test-AdminElevation**
- Returns [bool] indicating if current process is running as Administrator
- Uses [Security.Principal.WindowsPrincipal]::IsInRole("Administrator") pattern
- No parameters required
- Returns $true if elevated, $false otherwise

**Function 2: Invoke-AdminRelaunch**
- Parameter: [string[]]$KnownArgs = @() (passed known arguments from original invocation)
- Shows descriptive message: "WinOptimizer requires Administrator privileges. Restarting with elevation..." in Yellow
- Relaunches script using Start-Process with -Verb RunAs
- Uses powershell.exe with arguments: -NoProfile, -ExecutionPolicy Bypass, -File, $PSCommandPath

- Logs elevation relaunch action via Write-OptLog with Module="SafetyGates", Operation="ElevationRelaunch"
- Exits with code 0 after successful relaunch (original process terminates)
- On failure, exits with code 5 (ERROR_ACCESS_DENIED) and logs error via Write-OptLog

**Critical requirements from 02-CONTEXT.md:**
- Use [Security.Principal.WindowsPrincipal]::IsInRole("Administrator") for detection
- Use Start-Process with -Verb RunAs for relaunch

- Use -NoProfile and -ExecutionPolicy Bypass switches
- Show descriptive message before relaunch in Yellow color
- Exit code 5 if elevation fails or user cancels UAC
- Log elevation check to JSONL

<task type="auto">
  <name>Task 2: Implement Test-PowerShellVersion library helper</name>
  <files>lib/Test-PowerShellVersion.ps1</files>
  <action>
Create lib/Test-PowerShellVersion.ps1 with one function following Phase 1 library patterns:

**Function: Test-PowerShellVersion**
- Returns [hashtable] with keys: Passed (bool), Version (System.Version)
- No parameters required
- Detects PowerShell version via $PSVersionTable.PSVersion
- Handles three scenarios:

1. **PowerShell 7+ (pwsh)**: Major version >= 7
   - Shows interactive prompt: "Detected PowerShell X.X. This script is designed for PowerShell 5.1. Proceed? (Y/N)"
   - If user enters 'Y' or 'y': Returns @{ Passed = $true; Version = $currentVersion }
   - If user enters anything else: Logs error via Write-OptLog and exits with code 1
   - Logs warning via Write-OptLog: "Proceeding with PowerShell 7+ despite design target"

2. **PowerShell < 5.1**: Version less than 5.1
   - Shows error in Red: "ERROR: PowerShell 5.1+ required. Current version: X.X is not supported. Exiting."
   - Logs error via Write-OptLog
   - Exits with code 1

3. **PowerShell 5.1-6.x**: Version 5.1 to 6.x
   - Logs success via Write-OptLog: "PowerShell version validation passed"
   - Returns @{ Passed = $true; Version = $currentVersion }

**Critical requirements from 02-CONTEXT.md:**
- Allow PowerShell 7+ (pwsh) but show interactive prompt
- For PowerShell < 5.1: Fatal error with clear message
- Error messages should be clear and technical
- Log all version checks via Write-OptLog

**Follow Phase 1 patterns:**
- #Requires -Version 5.1 at file start
- Full comment-based help (.SYNOPSIS, .DESCRIPTION, .PARAMETER, .EXAMPLE, .NOTES)
- [CmdletBinding()] and [OutputType([hashtable])] attribute
- Try/catch error handling with Write-Error
- #region/#endregion blocks for organization
- No backtick line continuation
- Returns hashtable with structured data

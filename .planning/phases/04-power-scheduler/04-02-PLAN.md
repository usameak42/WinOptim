```powershell
#Requires -Version 5.1

function Invoke-SchedulerOptimize {
    [CmdletBinding()]
    [OutputType([bool])]
    param()

    # Dot-source lib helpers

    . $PSScriptRoot\..\lib\Save-RollbackEntry.ps1
    . $PSScriptRoot\..\lib\Get-ActivePlanGuid.ps1
```

**Begin Block:**
- Initialize counters: $successCount, $skipCount, $warningCount, $errorCount
- Initialize stopwatch: [System.Diagnostics.Stopwatch]::StartNew()
- Set global module: $global:CurrentModule = "Invoke-SchedulerOptimize"
- Display module header with description

**Process Block - Stage 1: Win32PrioritySeparation Tuning (SCHD-01)**
- Define registry path: $regPath = "HKLM:\SYSTEM\CurrentControlSet\Control\PriorityControl"
- Define value name: $valueName = "Win32PrioritySeparation"
- Define desired value: $desiredValue = 38
- Display explanation to user:
  - "[INFO] Win32PrioritySeparation=38 Configuration:"
  - "Value 38 enables:"
  - "- Variable quanta (dynamic CPU time allocation)"
  - "- Short intervals (faster context switching)"
  - "- 3x foreground boost (foreground apps get 3x more CPU time)"
  - "Result: Improved foreground application responsiveness"
- Prompt user: "Apply Win32PrioritySeparation=38? (Y/N)"
- If user accepts:
  - Check current value: $currentValue = Get-ItemProperty -Path $regPath -Name $valueName -ErrorAction SilentlyContinue
  - Idempotency check:
    - If $null -ne $currentValue -and $currentValue.$valueName -eq $desiredValue:
      - Write [SKIP] "Win32PrioritySeparation already set to $desiredValue"
      - Log SKIP: Write-OptLog with current/desired values
      - Increment $skipCount
      - Continue to next stage
  - Save original value: $originalValue = if ($null -ne $currentValue) { $currentValue.$valueName } else { $null }
  - Save rollback entry: Save-RollbackEntry -Type "Registry" -Target $regPath -ValueName $valueName -OriginalData $originalValue -OriginalType "REG_DWORD"
  - Set registry value: Set-ItemProperty -Path $regPath -Name $valueName -Value $desiredValue -Type DWord -ErrorAction Stop
  - Verify change: $newValue = Get-ItemProperty -Path $regPath -Name $valueName -ErrorAction Stop
  - If $newValue.$valueName -eq $desiredValue:
    - Write [SUCCESS] "Set Win32PrioritySeparation to $desiredValue"
    - Log SUCCESS: Write-OptLog with old/new values
    - Increment $successCount
  - Else:
    - Write [WARNING] "Win32PrioritySeparation value mismatch after set"
    - Log WARNING: Write-OptLog with expected/actual values
    - Increment $warningCount
  - On error:
    - Write [ERROR] with exception details
    - Log ERROR: Write-OptLog with error details
    - Increment $errorCount
- If user declines:
  - Write [SKIP] "User declined Win32PrioritySeparation tuning"
  - Log SKIP: Write-OptLog

**Process Block - Stage 2: CPU Core Parking Disablement (SCHD-03)**
- Define hardcoded GUIDs:
  - $processorSubGroup = "54533251-82be-4824-96c1-47b60b740d00"
  - $coreParkingSetting = "0cc5b647-c1df-4637-891a-dec35c318583"
  - $minProcessorState = "bc5038f7-23e0-4960-96da-33abaf5935ec"
  - $maxProcessorState = "3b04d4fd-1cc7-4f23-ab1c-d1337819c4bb"
- Get active plan GUID: $planGuid = Get-ActivePlanGuid
- Display explanation to user:
  - "[INFO] CPU Parking Explanation:"
  - "CPU parking puts CPU cores to sleep to save power."
  - "Disabling parking keeps all cores awake for maximum responsiveness."
  - "Trade-off: Improved responsiveness at cost of power efficiency."
- Prompt user: "Disable parking on: All cores / Logical cores only / AC power only / Skip? (A/L/O/S)"
- Handle user choice:
  - If "A" (All cores):
    - Set to 100%: powercfg /setacvalueindex $planGuid $processorSubGroup $coreParkingSetting 100
    - If LASTEXITCODE -eq 0:
      - Write [SUCCESS] "Disabled CPU core parking on all cores"
      - Log SUCCESS: Write-OptLog with value=100
      - Increment $successCount
    - Else:
      - Check if registry key exists: Test-Path "HKLM:\SYSTEM\CurrentControlSet\Control\Power\PowerSettings\$processorSubGroup\$coreParkingSetting"
      - If key doesn't exist:
        - Write [WARNING] "CPU core parking key not found on this system"
        - Log WARNING: Write-OptLog with "Registry key not found"
        - Increment $warningCount
      - Else:
        - Write [ERROR] with powercfg output
        - Log ERROR: Write-OptLog with error details
        - Increment $errorCount
  - If "L" (Logical cores only):
    - Set to 50%: powercfg /setacvalueindex $planGuid $processorSubGroup $coreParkingSetting 50
    - Log SUCCESS or ERROR similar to "A" case
  - If "O" (AC power only):
    - Set AC to 100%: powercfg /setacvalueindex $planGuid $processorSubGroup $coreParkingSetting 100
    - Set DC to 0%: powercfg /setdcvalueindex $planGuid $processorSubGroup $coreParkingSetting 0
    - Log SUCCESS or ERROR similar to "A" case
  - If "S" (Skip):
    - Write [SKIP] "User declined CPU core parking changes"
    - Log SKIP: Write-OptLog
- Save rollback entry before applying: Save-RollbackEntry -Type "Registry" -Target "$planGuid\$processorSubGroup" -ValueName $coreParkingSetting -OriginalData $originalValue -OriginalType "REG_DWORD"

**Process Block - Stage 3: Processor State Configuration (SCHD-04)**
- Use same hardcoded GUIDs from Stage 2
- Only execute if user didn't choose "S" (Skip) in Stage 2
- Set minimum processor state to 100% on AC:
  - Save rollback entry for current value
  - Apply: powercfg /setacvalueindex $planGuid $processorSubGroup $minProcessorState 100
  - If LASTEXITCODE -eq 0: Log SUCCESS, increment $successCount
  - Else: Log ERROR, increment $errorCount
- Set maximum processor state to 100% on AC:
  - Save rollback entry for current value
  - Apply: powercfg /setacvalueindex $planGuid $processorSubGroup $maxProcessorState 100
  - If LASTEXITCODE -eq 0: Log SUCCESS, increment $successCount
  - Else: Log ERROR, increment $errorCount
- Apply power settings: powercfg /SetActive $planGuid
- Note: Per CONTEXT.md, processor states apply to AC power only (preserve battery life on laptops)

**End Block:**
- Stop stopwatch
- Display summary with counters and elapsed time
- Return $true if $errorCount -eq 0, else $false

**CRITICAL REQUIREMENTS:**
- Use hardcoded GUIDs only (NO powercfg aliases)
- All registry operations must call Save-RollbackEntry BEFORE Set-ItemProperty
- All operations must call Write-OptLog AFTER execution with complete field population
- Idempotency checks: Skip if value already matches desired state
- User prompts per CONTEXT.md decisions (Win32PrioritySeparation, core parking options)
- Color-coded output: [SUCCESS] Green, [WARNING] Yellow, [ERROR] Red, [INFO] Cyan, [SKIP] Gray
- 4-space indentation with inline comments
- #region/#endregion blocks for each stage
- Error handling with try/catch blocks
- Handle missing registry keys gracefully (WARNING, continue)

  <name>Task 2: Implement network adapter interrupt moderation detection and configuration</name>
  <files>modules/Invoke-SchedulerOptimize.ps1</files>
  <action>
Add Stage 4 to Invoke-SchedulerOptimize.ps1 process block (after Stage 3):

**Process Block - Stage 4: Network Adapter Interrupt Moderation (SCHD-05)**

**Step 1: Detect Network Adapters**
- Query WMI for network adapters: $adapters = Get-WmiObject -Class Win32_NetworkAdapter -ErrorAction SilentlyContinue
- Filter for physical adapters only (exclude virtual adapters):
  - Exclude where Name contains "Virtual", "Hyper-V", "VMware", "VirtualBox"
  - Exclude where AdapterTypeId -ne 0 (Ethernet=0, Wireless=9, etc.)
- If no adapters found:
  - Write [INFO] "No physical network adapters detected"
  - Continue to end block
- Initialize array: $configuredAdapters = @()

**Step 2: Process Each Network Adapter**
- For each adapter in $adapters:
  - Extract adapter properties:
    - $adapterName = $adapter.Name
    - $adapterGUID = $adapter.GUID
    - $adapterType = if ($adapter.AdapterTypeId -eq 0) { "Ethernet" } elseif ($adapter.AdapterTypeId -eq 9) { "Wi-Fi" } else { "Other" }
  - Display adapter info: "[ADAPTER] $adapterName ($adapterType) - GUID: $adapterGUID"
  - Check for interrupt moderation registry key:
    - $regPath = "HKLM:\SYSTEM\CurrentControlSet\Control\Class\{4D36E972-E325-11CE-BFC1-08002BE10318}"
    - Search subkeys for matching adapter GUID
    - If matching key found:
      - Check for *InterruptModeration or *ITR value
      - $currentValue = Get-ItemProperty -Path $keyPath -Name "*InterruptModeration" -ErrorAction SilentlyContinue
    - If no registry key found:
      - Write [SKIP] "Interrupt moderation registry key not found for this adapter"
      - Log SKIP: Write-OptLog
      - Increment $skipCount
      - Continue to next adapter

**Step 3: Prompt User for Interrupt Moderation Settings**
- If interrupt moderation key exists:
  - Explain interrupt moderation: "Interrupt moderation reduces CPU overhead by coalescing network interrupts. Disabling can reduce latency but may increase CPU usage."
  - Display current value: "[CURRENT] Interrupt Moderation: $currentValue"
  - Prompt user: "Configure interrupt moderation for '$adapterName': Enable / Disable / Skip? (E/D/S)"
  - Handle user choice:
    - If "E" (Enable):
      - Set value to enable interrupt moderation (typically 1 or 2, varies by adapter)
      - Save rollback entry: Save-RollbackEntry -Type "Registry" -Target $keyPath -ValueName "*InterruptModeration" -OriginalData $currentValue -OriginalType "REG_DWORD"
      - Apply: Set-ItemProperty -Path $keyPath -Name "*InterruptModeration" -Value 1 -Type DWord -ErrorAction Stop
      - If success:
        - Write [SUCCESS] "Enabled interrupt moderation for '$adapterName'"
        - Log SUCCESS: Write-OptLog
        - Increment $successCount
        - Add to $configuredAdapters array
      - On error:
        - Write [ERROR] with exception details
        - Log ERROR: Write-OptLog
        - Increment $errorCount
    - If "D" (Disable):
      - Set value to disable interrupt moderation (typically 0)
      - Save rollback entry: Save-RollbackEntry -Type "Registry" -Target $keyPath -ValueName "*InterruptModeration" -OriginalData $currentValue -OriginalType "REG_DWORD"
      - Apply: Set-ItemProperty -Path $keyPath -Name "*InterruptModeration" -Value 0 -Type DWord -ErrorAction Stop
      - If success:
        - Write [SUCCESS] "Disabled interrupt moderation for '$adapterName'"
        - Log SUCCESS: Write-OptLog
        - Increment $successCount
        - Add to $configuredAdapters array
      - On error:
        - Write [ERROR] with exception details
        - Log ERROR: Write-OptLog
        - Increment $errorCount
    - If "S" (Skip):
      - Write [SKIP] "User declined interrupt moderation configuration for '$adapterName'"
      - Log SKIP: Write-OptLog
      - Increment $skipCount

**Step 4: Display Summary of Configured Adapters**
- If $configuredAdapters.Count -gt 0:
  - Write [INFO] "Configured interrupt moderation for $($configuredAdapters.Count) adapter(s):"
  - For each adapter in $configuredAdapters:
    - Write "  - $adapter"
- Else:
  - Write [INFO] "No network adapters configured for interrupt moderation"

**CRITICAL REQUIREMENTS:**
- Use WMI Get-WmiObject Win32_NetworkAdapter for detection (not wsl.exe)
- Filter out virtual adapters (Virtual, Hyper-V, VMware, VirtualBox)
- Search registry for interrupt moderation keys per adapter
- Interrupt moderation value varies by adapter (Enable=1 or 2, Disable=0)
- Prompt user per adapter for configuration choice
- Handle missing registry keys gracefully (SKIP, continue)
- Save rollback entry before modifying registry
- Log all operations with Write-OptLog
- Handle errors gracefully with try/catch
- Color-coded output per established patterns

**NOTE:** Interrupt moderation registry paths vary by adapter vendor (Intel, Realtek, Broadcom). The implementation should search for the key rather than assuming a specific path. If the key is not found, skip the adapter with a [SKIP] message.

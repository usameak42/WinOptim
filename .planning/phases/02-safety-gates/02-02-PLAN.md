Create lib/Invoke-RestorePoint.ps1 with one function following Phase 1 library patterns:

**Function: Invoke-RestorePointCreation**
- Parameter: [string]$RestorePointName = "WinOptimizer-Before-Optimization-$(Get-Date -Format 'yyyyMMdd')"
- Returns [bool] indicating success ($true) or failure ($false)
- Implements four-stage workflow:

**Stage 1: Check for recent restore points (within 1 hour)**
- Query existing restore points via Get-ComputerRestorePoint
- Sort by CreationTime descending
- Filter for points created within last hour: `Where-Object { $_.CreationTime -gt (Get-Date).AddHours(-1) }`
- If recent point found:
  - Show [SKIP] message in Cyan: "Recent restore point found: {Description} (created {CreationTime})"
  - Log skip via Write-OptLog with Result="Skip", Level="SKIP"
  - Return $true (no need to create new point)

**Stage 2: Check for old restore point (> 24 hours)**
- Filter for points created within last 24 hours: `Where-Object { $_.CreationTime -gt (Get-Date).AddHours(-24) }`
- If old point found:
  - Calculate age in hours: `((Get-Date) - $oldPoint.CreationTime).TotalHours`
  - Show [WARNING] message in Yellow: "Most recent restore point is {age} hours old."

**Stage 3: Create new restore point**
- Show [ACTION] message in Cyan: "Creating System Restore Point: {RestorePointName}"
- Show progress message: "This may take 10-30 seconds..." in Gray
- Use Checkpoint-Computer cmdlet:
  - Description: $RestorePointName parameter
  - RestorePointType: MODIFY_SETTINGS
  - ErrorAction: Stop (halt on failure)
- On success:
  - Show [SUCCESS] message in Green: "System Restore Point created successfully"
  - Log success via Write-OptLog with Module="SafetyGates", Operation="RestorePointCreate", Result="Success", Level="SUCCESS"
  - Update $global:RollbackData.RestorePoint hashtable with Name and Timestamp (ISO 8601 format)
  - Return $true
- On failure:
  - Show [ERROR] message in Red: "Failed to create System Restore Point: {error details}"
  - **CRITICAL**: Even in silent mode, prompt user interactively: "Restore Point creation failed. Continue anyway? (Y/N)"
  - If user enters 'Y' or 'y':
    - Log warning via Write-OptLog with Result="Warning", Level="WARNING"
    - Return $true (user chose to continue)
  - If user enters anything else:
    - Log error via Write-OptLog with Result="Error", Level="ERROR"
    - Exit with code 1

**Stage 4: Error handling**
- Wrap Get-ComputerRestorePoint query in try/catch
- If query fails, show [WARNING] and continue (don't halt)
- Wrap Checkpoint-Computer in try/catch with ErrorAction Stop
- Log all errors via Write-OptLog

```powershell
#Requires -Version 5.1

function Invoke-PowerPlanConfig {
    [CmdletBinding()]
    [OutputType([bool])]
    param()

    # Dot-source lib helpers
    . $PSScriptRoot\..\lib\Write-OptLog.ps1
    . $PSScriptRoot\..\lib\Save-RollbackEntry.ps1
    . $PSScriptRoot\..\lib\Get-ActivePlanGuid.ps1
```

**Begin Block:**
- Initialize counters: $successCount, $skipCount, $warningCount, $errorCount
- Initialize stopwatch: [System.Diagnostics.Stopwatch]::StartNew()
- Set global module: $global:CurrentModule = "Invoke-PowerPlanConfig"
- Display module header with description

**Process Block - Stage 1: Modern Standby (S0) Detection and Override (PWRP-01, PWRP-02)**
- Define registry path: $s0KeyPath = "HKLM:\SYSTEM\CurrentControlSet\Control\Power"
- Define value name: $s0ValueName = "PlatformAoAcOverride"
- Detect S0: Get-ItemProperty -Path $s0KeyPath -Name $s0ValueName -ErrorAction SilentlyContinue
- If S0 detected ($null -ne $s0Value):
  - Write [WARNING] explaining Modern Standby suppresses Ultimate Performance plan
  - Prompt user: "Apply S0 fix (requires reboot)? (Y/N)"
  - If user accepts:
    - Save rollback entry: Save-RollbackEntry -Type "Registry" -Target $s0KeyPath -ValueName $s0ValueName -OriginalData $s0Value.$s0ValueName -OriginalType "REG_DWORD"
    - Apply fix: Set-ItemProperty -Path $s0KeyPath -Name $s0ValueName -Value 0 -Type DWord
    - Log operation: Write-OptLog with WARNING level
    - Prompt reboot timing: "Reboot now? (Y/N)"
    - If "Y": Restart-Computer -Force
    - If "N": Write [WARNING] that reboot is required for fix to take effect
  - If user declines: Log SKIP message
- If S0 not detected: Skip silently (per CONTEXT.md decision)

**Process Block - Stage 2: Power Plan Duplication and Activation (PWRP-03, PWRP-04)**
- Define constants:
  - $ultimatePerfGuid = "e9a42b02-d5df-448d-aa00-03f14749eb61"
  - $highPerfGuid = "8c5e7fda-e8bf-45a6-a6cc-4b3c3f300d00" (fallback)
  - $customPlanName = "WinOptimizer Ultimate"
- Check for existing plan: powercfg /list | Select-String -Pattern $customPlanName
- If plan exists:
  - Extract existing GUID via regex
  - Prompt user: "Reuse existing / Delete and recreate / Cancel? (R/D/C)"
  - Handle "R": Set $planGuid = existingGuid, continue
  - Handle "D": powercfg /delete $existingGuid, continue to create new
  - Handle "C": Return $false (user cancelled)
- If plan doesn't exist:
  - Duplicate Ultimate Performance: powercfg /duplicatescheme $ultimatePerfGuid
  - If LASTEXITCODE -ne 0 (Ultimate Performance not available):
    - Write [WARNING] about falling back to High Performance
    - Duplicate High Performance: powercfg /duplicatescheme $highPerfGuid
    - If still fails: Write [ERROR] and return $false
  - Extract new GUID from output via regex: ([a-f0-9]{8}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{12})
  - Rename plan: powercfg /changename $planGuid $customPlanName
  - If rename fails: Log WARNING (non-critical)
- Activate plan: powercfg /setactive $planGuid
- Verify activation: Call Get-ActivePlanGuid and compare with $planGuid
- If match: Write [SUCCESS], log operation, increment $successCount
- If mismatch: Write [WARNING], log verification failure, increment $warningCount

**Process Block - Stage 3: PCIe and USB Power Settings (PWRP-05, PWRP-06)**
- Define hardcoded GUIDs:
  - $pciSubGroup = "501a4d13-42af-4429-9fd1-a8218c268e20" (PCI Express)
  - $pciLinkStatePower = "12c0bdb0-2d34-4600-9843-91eb582ae84f" (Link State Power Management)
  - $usbSubGroup = "2a737441-1930-4402-8d77-b2bebba308a3" (USB)
  - $usbSelectiveSuspend = "48e6b7a6-50f5-4782-a5d4-53bb8f07e226" (USB Selective Suspend)
- Get active plan GUID: $planGuid = Get-ActivePlanGuid
- Configure PCIe Link State Power Management to Off:
  - Save rollback entry for current value
  - Apply: powercfg /setacvalueindex $planGuid $pciSubGroup $pciLinkStatePower 0
  - If LASTEXITCODE -eq 0: Log SUCCESS, increment $successCount
  - Else: Log ERROR, increment $errorCount
- Configure USB Selective Suspend to Disabled:
  - Save rollback entry for current value
  - Apply: powercfg /setacvalueindex $planGuid $usbSubGroup $usbSelectiveSuspend 0
  - If LASTEXITCODE -eq 0: Log SUCCESS, increment $successCount
  - Else: Log ERROR, increment $errorCount
- Apply power settings: powercfg /SetActive $planGuid

**End Block:**
- Stop stopwatch
- Display summary with counters and elapsed time
- Return $true if $errorCount -eq 0, else $false

- All registry operations must call Save-RollbackEntry BEFORE Set-ItemProperty

Add Stage 4 to Invoke-PowerPlanConfig.ps1 process block (after Stage 3):

**Process Block - Stage 4: OEM Service Detection and Countermeasures (PWRP-07, PWRP-08)**

**Step 1: Load OEM Service Configuration**
- Define config path: $configPath = "$PSScriptRoot\..\config\services.json"
- Load config: $servicesConfig = Get-Content -Path $configPath | ConvertFrom-Json
- Initialize array: $oemServicesDetected = @()

**Step 2: Detect OEM Power Services**
- Iterate through $servicesConfig.oem.PSObject.Properties:
  - Extract vendor name: $vendorName = $vendor.Name
  - Extract vendor services: $vendorServices = $vendor.Value
  - For each service in $vendorServices:
    - Extract service properties: $serviceName, $displayName, $detectionPattern
    - Execute detection query: $serviceExists = Invoke-Expression $detectionPattern.query
    - If $serviceExists (not null):
      - Add to $oemServicesDetected array with Vendor, Name, DisplayName

**Step 3: Handle Detected OEM Services**
- If $oemServicesDetected.Count -gt 0:
  - Display [WARNING] header listing detected services
  - For each service in $oemServicesDetected:
    - Write service details: "[$vendorName] $displayName ($serviceName)"
  - Display explanation: "OEM services may reassert OEM power plans at boot/login."
  - Prompt user: "Disable detected OEM services? (Y/N)"
  - If user accepts:
    - For each service in $oemServicesDetected:
      - Get service object: $svc = Get-Service -Name $service.Name -ErrorAction Stop
      - Save rollback entry: Save-RollbackEntry -Type "Service" -Target $service.Name -OriginalStartType $svc.StartType
      - Stop service: Stop-Service -Name $service.Name -Force -ErrorAction Stop
      - Disable service: Set-Service -Name $service.Name -StartupType Disabled -ErrorAction Stop
      - Log SUCCESS: Write-OptLog with service details
      - Increment $successCount
    - On error: Write [ERROR], log failure, increment $errorCount
  - If user declines: Write [SKIP], log user declined

**Step 4: Scheduled Task Creation for OEM Countermeasures**
- After OEM services disabled (if user accepted):
  - Prompt user: "Create scheduled task to reapply power plan at login? (Y/N)"
  - Explain purpose: "Task will reactivate WinOptimizer Ultimate plan at login to counter OEM interference."
  - If user accepts:
    - Get active plan GUID: $planGuid = Get-ActivePlanGuid
    - Define task properties:
      - $taskName = "WinOptimizer Power Plan Reapply"
      - $taskDescription = "Reapply WinOptimizer Ultimate power plan at login to counter OEM interference"
    - Check for existing task: Get-ScheduledTask -TaskName $taskName -ErrorAction SilentlyContinue
    - If task exists:
      - Write [WARNING] that task already exists
      - Prompt user: "Overwrite existing task? (Y/N)"
      - If user accepts: Unregister-ScheduledTask -TaskName $taskName -Confirm:$false
      - If user declines: Return (keep existing task)
    - Create task action: $action = New-ScheduledTaskAction -Execute "powercfg.exe" -Argument "/setactive $planGuid"
    - Create trigger: $trigger = New-ScheduledTaskTrigger -AtLogon
    - Create principal: $principal = New-ScheduledTaskPrincipal -UserId $env:USERNAME -RunLevel Highest -LogonType Interactive
    - Create settings: $settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable
    - Register task: Register-ScheduledTask -TaskName $taskName -Action $action -Trigger $trigger -Principal $principal -Settings $settings -Description $taskDescription -ErrorAction Stop
    - If success:
      - Write [SUCCESS] with task name
      - Log operation: Write-OptLog with task details
      - Save rollback entry: Save-RollbackEntry -Type "ScheduledTask" -Target $taskName -OriginalData $null
      - Increment $successCount
    - On error:
      - Write [ERROR] with exception details
      - Log failure: Write-OptLog with error details
      - Increment $errorCount
  - If user declines: Write [SKIP], log user declined

**Step 5: No OEM Services Detected**
- If $oemServicesDetected.Count -eq 0:
  - Write [INFO] "No OEM power management services detected"
  - Continue to Stage 5 (if exists) or end block

**CRITICAL REQUIREMENTS:**
- Load config/services.json from relative path
- Iterate through all vendors (asus, lenovo, dell, hp)
- Execute detectionPattern.query for each service
- Prompt user before disabling OEM services
- Prompt user before creating scheduled task
- Create scheduled task with AtLogon trigger
- Task should run with highest privileges (RunLevel Highest)
- Task should allow running on battery (AllowStartIfOnBatteries, DontStopIfGoingOnBatteries)
- Save rollback entry for scheduled task creation
- Log all operations with Write-OptLog
- Handle errors gracefully with try/catch
- Color-coded output per established patterns
